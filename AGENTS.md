# AGENTS.md - 将来残高シミュレーター開発ロードマップ

このドキュメントは、AI開発アシスタント（Codex、Cursor、Cline、Roo Code等）が、ユーザーの要望に基づいた「将来の資金繰り（キャッシュフロー）シミュレーター」を、ゼロコストかつ最速で開発するための指示書・設計図である。

---

## 1. プロジェクト概要

### 1.1 アプリケーション名（仮）

- **MiraiZandaka (未来残高シミュレーター)**

### 1.2 解決する課題

- 毎月の固定費や不定期な支出（車検、税金など）がある中で、「3ヶ月後や半年後に手元にいくら残るのか」を電卓で何度も計算するのが面倒。
- 未来の残高を可視化し、資金ショートの不安を解消、または早期に手を打てるようにする。

### 1.3 コスト・収益方針

- **ランニングコスト**: 完全0円（サーバー代、データベース代、ドメイン代をかけない）。
- **収益化（将来性）**: 画面下部等へのWeb広告（Google AdSense等）の設置スペースをあらかじめ考慮する。

---

## 2. システムアーキテクチャ（ゼロコスト構成）

AIエージェントは以下の「サーバーレス・フロントエンド完結型」の構成で実装を行うこと。

- **プラットフォーム**: Webアプリケーション（スマートフォン・PCの両方に対応するレスポンシブWebデザイン）
- **ホスティング**: Vercel または Netlify (無料プラン枠を想定)
- **開発言語/環境**:
  - HTML5
  - CSS3 (Tailwind CSSをCDN経由で利用し、軽量かつ迅速にデザイン)
  - JavaScript (Vanilla JS、外部フレームワークを使わない、またはVue.js等のCDN版でシンプルに構築)
  - グラフ描画: Chart.js (CDN経由)
- **データ保存**: ブラウザの `localStorage` を使用（サーバー用データベースは使用しない）。

---

## 3. エージェントの役割定義（Roles）

AIが開発を進める際は、以下の役割を切り替えて思考・実装すること。

### @Product-Manager

- ユーザーの「電卓の手間を減らす」という本質的なニーズからブレないよう、機能をシンプルに保つ（機能の肥大化を防ぐ）。
- 収益化のための広告スペースをデザインに自然に組み込む。

### @Architect

- 外部サーバーに依存しない、`localStorage` を用いた堅牢なデータ保存設計を行う。
- 拡張性を考慮し、データ構造をシンプルに定義する。

### @UI-UX-Designer

- 「電卓感覚」で迷わず入力できるUI。スマホでの片手操作を意識したボタン配置。
- 未来の残高が「増えているか・減っているか」が直感的にわかるカラーユニバーサルデザイン。

### @Developer

- 未来の残高を日ごと、あるいは月ごとにシミュレーション計算するロジックを正確にJavaScriptで実装する。
- バグの混入を防ぐため、フェーズごとに動作確認が可能なクリーンコードを書く。
